# UltraReader Product Context

## Register

product

## Product Purpose

UltraReader is a local macOS documentation reader for Markdown files. It scans a chosen folder, shows the file structure as a navigable tree, renders selected Markdown through Quarkdown, and presents the result as a focused reading surface.

## Users

- Developers reading local project notes, specs, CLI docs, handoffs, and README files.
- Power users who keep large workspaces of Markdown documents and need fast browsing without leaving a native desktop app.
- The primary user is working on a Mac with many files open and wants a calm, scannable reading tool rather than a heavy editor.

## Tone

Clear, competent, quiet, and polished. The app should feel like a premium documentation reader: practical first, but with enough visual character to avoid looking like a raw file browser.

## Strategic Principles

- Reading is the main task. Navigation should support reading without dominating the window.
- Preserve local file structure, but do not make the UI feel like Finder.
- Quarkdown rendering should look like intentional documentation, not an unstyled web page.
- Prefer native macOS affordances where useful, but the visual result should feel more crafted than a default utility window.
- Avoid loud saturation across the whole surface. Use warm orange as a focused accent, not as background noise.

## Anti-References

- Pale blue plus yellow combinations that feel accidental or low contrast.
- Oversized typography that makes normal docs feel zoomed-in.
- Double borders around code blocks or nested panels.
- Raw LaTeX-like document styling for everyday Markdown.
- Generic sidebar/file tree chrome that overwhelms the reading pane.
